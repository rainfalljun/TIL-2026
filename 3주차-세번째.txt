

# 주요 개념 설명

GitHub (원격 저장소)
내 컴퓨터가 아닌 인터넷상에 내 코드를 저장해 두는 안전한 창고라고 생각하면 됩니다. 여러 사람이 함께 작업할 때 사용합니다.

Push (밀어넣기)
내 컴퓨터에서 작업한 내용을 인터넷 창고인 GitHub로 업로드하는 과정입니다.

Pull (당겨오기)
인터넷 창고인 GitHub에 새로 올라온 내용을 내 컴퓨터로 내려받는 과정입니다.

-----

# 처음 시작할 때 순서대로 따라하기

새 프로젝트를 만들고 인터넷 창고(GitHub)와 연결하는 방법입니다.

1단계: Git 시작하기
터미널이나 명령 프롬프트를 열고 아래 명령어를 입력하세요. 현재 폴더를 Git이 관리하도록 설정합니다.
git init

2단계: 설명 파일 만들기
프로젝트를 소개하는 파일을 하나 만듭니다.
echo it\_lecture \> README.md

3단계: 파일 장바구니에 담기
방금 만든 파일을 기록하기 위해 장바구니에 넣는 과정입니다.
git add README.md

4단계: 기록 남기기
장바구니에 담긴 파일들을 확정 지어 기록합니다. 첫 번째 기록이라는 메시지를 남깁니다.
git commit -m first commit

5단계: 기본 통로 이름 정하기
기본적인 작업 통로의 이름을 main으로 설정합니다.
git branch -M main

6단계: 인터넷 창고 주소 알려주기
내 컴퓨터에게 데이터를 보낼 GitHub 주소를 알려줍니다.
git remote add origin [https://github.com/사용자아이디/저장소이름.git](https://www.google.com/search?q=https://github.com/%EC%82%AC%EC%9A%A9%EC%9E%90%EC%95%84%EC%9D%B4%EB%94%94/%EC%A0%80%EC%9E%A5%EC%86%8C%EC%9D%B4%EB%A6%84.git)

7단계: 파일 업로드하기
인터넷 창고로 파일을 처음 보냅니다.
git push -u origin main

-----

# 일상적인 작업 순서 (업데이트할 때)

평소에 코드를 수정하고 업로드할 때 반복하는 과정입니다.

1단계: 변경된 모든 파일 담기
수정한 파일들을 모두 장바구니에 넣습니다.
git add .

2단계: 기록하기
수정 내용이 무엇인지 짧게 적어 기록을 남깁니다.
git commit -m 수정내용작성

3단계: 업로드하기
GitHub로 최종 전송합니다.
git push origin main

-----

# 주의사항 (인증 관련)

GitHub에 파일을 올릴 때 비밀번호를 물어보는 경우가 있습니다. 이때 우리가 사용하는 일반 로그인 비밀번호가 아니라 GitHub 사이트에서 따로 발급받은 Access Token (액세스 토큰) 코드를 복사해서 붙여넣어야 로그인이 완료됩니다.
