
[7주차 핵심 개념 요약 정리]

1. 서브에이전트 (Sub-agents)
- 메인 AI 외에 특정 역할만 수행하도록 전문화시킨 보조 AI들입니다.
- 코드 리뷰어, 성능 최적화 전문가, UX 개선 전문가 등으로 역할을 나눕니다.
- 팀장 역할을 하는 메인 에이전트가 서브에이전트들에게 업무를 지시하고 보고받는 방식으로 협업합니다.
- 복잡한 프로젝트를 작게 쪼개어 정교하게 작업할 수 있다는 장점이 있습니다.

2. MCP (Model Context Protocol)
- AI가 외부 데이터나 도구에 직접 연결될 수 있게 해주는 표준 규격입니다.
- AI가 내 컴퓨터의 파일을 읽거나 수정하고, 노션이나 구글 드라이브 같은 외부 서비스의 데이터를 실시간으로 가져올 수 있게 합니다.
- 사람이 일일이 코드를 짜서 전달해야 하는 API와 달리, 규격만 맞으면 AI가 스스로 필요한 도구를 골라 사용하는 능동적인 방식입니다.

3. 주요 MCP 도구 활용
- Notion MCP: 노션 페이지를 읽거나, 조사한 내용을 자동으로 노션 노트에 기록합니다.
- Sequential Thinking MCP: 복잡한 문제를 해결할 때 AI가 단계별로 생각을 정리하며 추론하게 하여 정확도를 높입니다.
- Playwright MCP: 브라우저를 자동으로 실행해 웹사이트가 잘 작동하는지 테스트합니다.

4. 터미널 주요 명령어
- 서버 추가: claude mcp add --transport http [이름] [주소]
- 목록 확인: /mcp 또는 claude mcp list
- 에이전트 관리: /agents

5. 이번 주 학습 목표 체크리스트
- 서브에이전트 3종(리뷰, 최적화, UX) 생성 및 역할 이해
- AI 개발팀 구성을 통한 프로젝트 분석 및 개선 리포트 확인
- 노션 MCP 연결 및 리서치 노트 자동 생성 실습
- Sequential Thinking MCP를 활용한 추론 성능 비교
- Playwright MCP를 이용한 앱 자동 테스트 수행
