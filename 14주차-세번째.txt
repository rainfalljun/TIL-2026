========================================
13장. 다국어 처리 (웹자바프로그래밍 응용)
========================================

[학습목표]
- 국제화의 개념에 대해 학습한다.
- 리소스 파일을 이용한 다국어 처리 방법에 대해 학습한다.


1. 다국어 처리의 개요
----------------------------------------
■ 다국어 처리
- 한 웹 애플리케이션이 여러 언어와 지역별로 다른 표현 형식을 지원하는 것
- 웹 브라우저의 로케일에 따라 다양한 언어를 지원하는 서비스

■ 로케일(Locale)
- 사용자의 언어, 국가, 시간대 등 사용자가 선호하는 사항을 지정한 매개변수 모음
- 코드를 수정하지 않고도 접속한 사용자의 언어/지역 차이를 반영해
  자동으로 메시지를 변경하는 기능

■ 국제화(Internationalization, I18n)와 현지화(Localization, L10n)
- 국제화: 코드베이스를 크게 수정하지 않고 다양한 언어와 지역에
  서비스할 수 있도록 소프트웨어를 설계하는 과정
- 현지화(지역화): 텍스트를 번역하고 로케일별 구성 요소를 추가하여
  국제화된 소프트웨어를 특정 지역/언어에 맞게 조정하는 과정
- 적용 범위: 문자 집합(유니코드), 측정 단위(통화, 온도, 거리),
  시간 및 날짜 형식, 텍스트 방향 등

■ Spring에서의 다국어 처리 구성 요소
- MessageSource: 언어별 메시지 코드와 메시지를 정의하여
  화면 출력 메시지를 다국어로 표현
- LocaleResolver: 현재 로케일을 결정하고 저장
- LocaleChangeInterceptor: 사용자가 원하는 언어를 선택/변경할 수 있도록 지원


2. MessageSource를 이용한 다국어 처리
----------------------------------------
■ 메시지 리소스(ResourceBundle) 파일 작성
- 형식: 파일이름[_언어코드[_국가코드]].properties
- key=value 쌍으로 구성 (key는 뷰 페이지에서 메시지 참조에 사용, 보통 소문자)
- 위치: src/main/resources 폴더

  파일 이름 규칙
  - 파일이름.properties        : 일치하는 리소스 파일이 없을 때 사용(기본)
  - 파일이름_ko.properties     : 시스템 언어가 한국어일 때
  - 파일이름_en.properties     : 시스템 언어가 영어일 때
  - 파일이름_en_UK.properties  : 영어 + 국가코드가 영국일 때
  - 파일이름_ja.properties     : 시스템 언어가 일본어일 때

- *.properties 파일은 기본 ISO-8859-1 인코딩이지만 UTF-8도 지원 가능
  (ISO-8859-1 형식에서 한글은 유니코드로 변환되어 저장됨)
- 예: person.form.enter.message=당신의 정보를 입력하세요

■ MessageSource 환경 설정 (servlet-context.xml)
<beans:bean id="messageSource"
    class="org.springframework.context.support.ResourceBundleMessageSource">
    <beans:property name="basename" value="messages" />
    <beans:property name="defaultEncoding" value="UTF-8" />
</beans:bean>

  MessageSource 구현체
  - ResourceBundleMessageSource
    : ResourceBundle, MessageFormat 클래스 기반.
      한번 로드된 메시지는 캐시되어 재사용됨.
  - ReloadableResourceBundleMessageSource
    : cacheSeconds 프로퍼티 설정으로 재시작 없이
      실행 도중 메시지를 다시 로드할 수 있음.

■ 뷰 페이지에 메시지 출력
- 태그 라이브러리 선언:
  <%@ taglib prefix="spring" uri="http://www.springframework.org/tags" %>
- <spring:message> 태그로 메시지 출력
  예: <spring:message code="addBook.form.title.label"/>

  주요 속성
  - code      : 출력할 메시지의 키
  - text      : code에 대한 메시지를 찾을 수 없을 때 출력할 기본 텍스트
  - arguments : 메시지 출력을 위한 인자 전달
  - argumentSeparator : 인자 구분자 (기본값: 콤마)
  - var       : 결과를 변수로 바인딩할 때 사용할 변수명
  - scope     : var 사용 시 변수 적용 범위 (page/request/session/application)
  - htmlEscape / javaScriptEscape : 문자열 형식 변환 (기본값 false)
  - message   : 유효성 검사 오류 메시지를 간단히 보여줄 때 사용

■ 메시지 출력에 인자 사용하기
- 메시지에서 인자 삽입 위치를 {n} 형식으로 선언 (n은 0부터 시작)
- 언어에 따라 출력되는 파라미터 순서가 달라질 수 있음
- 예)
  메시지 정의: addBook.form.param=파라미터 {0} 와 {1}
  JSP 출력  : <spring:message code="addBook.form.param" arguments="Banana,Apple"/>
  결과      : 파라미터 Banana 와 Apple


3. LocaleResolver와 LocaleChangeInterceptor를 이용한 다국어 변경
----------------------------------------
■ 요청 처리 흐름
1) LocaleResolver가 환경 정보로 로케일 결정 (resolveLocale)
2) LocaleChangeInterceptor가 요청 파라미터로 로케일 변경 (setLocale)
3) 설정된 저장소(쿠키/세션 등)에 로케일 저장

■ LocaleResolver
- 환경정보를 이용해 로케일을 결정하고 저장하는 Spring 인터페이스

  구현체 종류
  - AcceptHeaderLocaleResolver : 브라우저의 accept-language 헤더 사용 (기본)
  - CookieLocaleResolver       : 쿠키에 로케일 정보 유지
  - SessionLocaleResolver      : 세션의 locale 속성 사용
  - FixedLocaleResolver        : 항상 고정된 로케일 반환

- CookieLocaleResolver 설정 예:
<beans:bean id="localeResolver"
    class="org.springframework.web.servlet.i18n.CookieLocaleResolver">
    <beans:property name="defaultLocale" value="ko" />
    <beans:property name="cookieName" value="clientLanguage" />
    <beans:property name="cookieMaxAge" value="100000" />
</beans:bean>

■ LocaleChangeInterceptor
- 별도의 컨트롤러 구현 없이 웹 요청의 매개변수만으로 로케일 변경 가능
<interceptors>
    <beans:bean class="org.springframework.web.servlet.i18n.LocaleChangeInterceptor">
        <beans:property name="paramName" value="language" />
    </beans:bean>
</interceptors>


4. 실습 요약
----------------------------------------
① MessageSource를 이용한 다국어 처리
- src/main/resources에 messages_ko.properties, messages_en.properties 작성
  (작성 전 파일 인코딩을 UTF-8로 변경)
- servlet-context.xml에 ResourceBundleMessageSource 빈 등록
  (basename="messages", defaultEncoding="UTF-8")
- addBook.jsp에서 하드코딩된 텍스트를 <spring:message code="..."/> 태그로 교체
- 테스트: http://localhost:8080/books/add 접속 후
  브라우저 언어 설정을 영어로 변경하여 메시지 변경 확인

② LocaleResolver + LocaleChangeInterceptor를 이용한 언어 변경
- servlet-context.xml에 등록:
  · LocaleChangeInterceptor (paramName="language")
  · SessionLocaleResolver (defaultLocale="ko")
- addBook.jsp에 언어 선택 링크 추가:
  <a href="?language=ko">Korean</a> | <a href="?language=en">English</a>
- 테스트: http://localhost:8080/books/add 접속 후
  Korean | English 링크 클릭으로 언어 변경 확인


[핵심 정리]
- 다국어 처리 = MessageSource(메시지 정의) + LocaleResolver(로케일 결정/저장)
  + LocaleChangeInterceptor(로케일 변경)
- 메시지 리소스 파일은 key=value 형식, 파일명에 언어/국가 코드 포함
- 뷰에서는 <spring:message code="키"/> 태그로 출력
- URL 파라미터(?language=en)만으로 간편하게 언어 전환 가능
