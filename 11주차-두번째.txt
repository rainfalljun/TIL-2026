1. 큐(Queue)의 개념과 구조
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 정의
  - 삽입과 삭제의 위치가 제한된 유한 순서 리스트
  - 뒤(rear)에서 삽입, 앞(front)에서 삭제만 가능한 구조
  - 선입선출(FIFO, First-In-First-Out) 구조

■ 스택 vs 큐 비교
  ┌────────┬──────────┬──────────┬──────────┬──────────┐
  │ 자료구조│  삽입연산 │  삽입위치 │  삭제연산 │  삭제위치 │
  ├────────┼──────────┼──────────┼──────────┼──────────┤
  │  스택  │   push   │   top    │   pop    │   top    │
  │  큐    │ enQueue  │   rear   │ deQueue  │  front   │
  └────────┴──────────┴──────────┴──────────┴──────────┘

■ 큐의 추상 자료형(ADT)
  - createQueue()  : 공백 큐 생성
  - isQueueEmpty() : 공백 상태 검사
  - enQueue(Q, item): rear에 원소 삽입
  - deQueue(Q)     : front 원소 삭제 및 반환
  - peek(Q)        : front 원소 반환 (삭제 없음)

■ 큐의 연산 과정 예시
  ① createQueue() → front = rear = -1
  ② enQueue(Q, A) → [A]         front=0, rear=0
  ③ enQueue(Q, B) → [A, B]      front=0, rear=1
  ④ deQueue(Q)   → [B]    반환:A front=1, rear=1
  ⑤ enQueue(Q, C) → [B, C]      front=1, rear=2
  ⑥ deQueue(Q)   → [C]    반환:B
  ⑦ deQueue(Q)   → []     반환:C

2. 큐의 구현
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

─────────────────────────────────────────
[1] 순차 큐 (1차원 배열 이용)
─────────────────────────────────────────
■ 상태 표현
  - 초기 상태 : front = rear = -1
  - 공백 상태 : front == rear
  - 포화 상태 : rear == n-1

■ 주요 알고리즘
  · 생성 : front ← -1; rear ← -1
  · 삽입 : rear ← rear + 1; Q[rear] ← item
  · 삭제 : front ← front + 1; return Q[front]
  · 검색 : return Q[front + 1]

■ 문제점
  - 삽입/삭제 반복 시 앞 공간이 비어있어도 rear=n-1이면 포화로 오인
  - 해결책1: 원소를 앞으로 이동 (비효율적)
  - 해결책2: 원형 큐 사용 ✓

─────────────────────────────────────────
[2] 원형 큐 (Circular Queue)
─────────────────────────────────────────
■ 개념
  - 1차원 배열의 처음과 끝이 논리적으로 연결된 구조
  - 나머지 연산자(mod)를 사용하여 인덱스 순환

■ 상태 표현
  - 초기 상태 : front = rear = 0
  - 공백 상태 : front == rear
  - 포화 상태 : (rear + 1) mod n == front
  ※ front 자리는 항상 빈자리로 유지

■ 순차 큐 vs 원형 큐 비교
  ┌────────┬───────────────────┬───────────────────┐
  │ 종류   │     삽입 위치      │     삭제 위치      │
  ├────────┼───────────────────┼───────────────────┤
  │ 순차 큐│   rear = rear + 1  │  front = front + 1 │
  │ 원형 큐│ rear=(rear+1) mod n│front=(front+1) modn│
  └────────┴───────────────────┴───────────────────┘

■ 주요 알고리즘
  · 생성 : front ← 0; rear ← 0
  · 삽입 : rear ← (rear+1) mod n; cQ[rear] ← item
  · 삭제 : front ← (front+1) mod n; return cQ[front]

─────────────────────────────────────────
[3] 연결 큐 (Linked Queue)
─────────────────────────────────────────
■ 개념
  - 단순 연결 리스트를 이용한 큐
  - front: 첫 번째 노드를 가리키는 포인터
  - rear : 마지막 노드를 가리키는 포인터

■ 상태 표현
  - 초기/공백 상태 : front = rear = NULL

■ 주요 알고리즘
  · 생성 : front ← NULL; rear ← NULL
  · 삽입 : 새 노드 생성 → 공백이면 front/rear 모두 새 노드 지정
            공백 아니면 rear.link ← new; rear ← new
  · 삭제 : old ← front; front ← front.link
            공백되면 rear ← NULL; 노드 반환
  · 검색 : return front.data

■ 장점
  - 크기 제한 없음 (동적 메모리 할당)
  - 포화 상태 문제 없음

3. 데크 (Deque: Double-Ended Queue)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 정의
  - 큐 두 개 중 하나를 뒤집어 붙인 구조
  - 양쪽 끝(front, rear) 모두에서 삽입/삭제 가능

■ 주요 연산
  - insertFront(DQ, item) : front 앞에 삽입
  - insertRear(DQ, item)  : rear 뒤에 삽입
  - deleteFront(DQ)       : front 원소 삭제 및 반환
  - deleteRear(DQ)        : rear 원소 삭제 및 반환
  - getFront(DQ)          : front 원소 반환
  - getRear(DQ)           : rear 원소 반환

■ 구현 방법
  - 순차 자료구조는 비효율적
  - 이중 연결 리스트(doubly linked list) 사용 권장
    → 각 노드: llink | data | rlink 구조

■ 연산 과정 예시
  ① createDeque()
  ② insertFront(DQ, 'A') → [A]
  ③ insertFront(DQ, 'B') → [B, A]
  ④ insertRear(DQ,  'C') → [B, A, C]
  ⑤ deleteFront(DQ)      → [A, C]    반환:B
  ⑥ deleteRear(DQ)       → [A]       반환:C
  ⑦ insertRear(DQ,  'D') → [A, D]
  ⑧ insertFront(DQ, 'E') → [E, A, D]
  ⑨ insertFront(DQ, 'F') → [F, E, A, D]

4. 큐의 응용
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 운영체제의 작업 큐
  · 프린터 버퍼 큐 (Printer Buffer Queue)
    - CPU → 프린터로 보낸 데이터를 FIFO 순서로 출력
  · 스케줄링 큐 (Scheduling Queue)
    - CPU 사용 요청 프로세스 순서 관리
    - 준비 큐(Ready Queue), 대기 큐(Waiting Queue) 활용

■ 시뮬레이션 (큐잉 시스템)
  · 큐잉 이론(Queue Theory) 사용
  · 대기행렬, 대기시간 등을 수학적으로 모델링

================================================================
                          핵심 요약
================================================================
 구분     │ 순차 큐  │ 원형 큐         │ 연결 큐
──────────┼──────────┼─────────────────┼──────────────────
 자료구조  │ 1차원배열 │ 1차원배열(순환)  │ 단순 연결 리스트
 초기상태  │ f=r=-1   │ f=r=0           │ f=r=NULL
 공백조건  │ f == r   │ f == r          │ f == NULL
 포화조건  │ r == n-1 │ (r+1)%n == f    │ 없음 (동적할당)
 삽입위치  │ rear+1   │ (rear+1) mod n  │ rear 다음 노드
 삭제위치  │ front+1  │ (front+1) mod n │ front 노드
