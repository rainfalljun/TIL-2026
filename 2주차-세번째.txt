

1. 계층적 구조의 정의
전체 애플리케이션을 역할과 책임에 따라 여러 개의 층으로 나누어 구성하는 것을 의미합니다. 이렇게 나누면 코드가 깔끔해지고 나중에 수정하거나 기능을 추가하기가 훨씬 쉬워집니다.

2. 주요 계층별 역할
애플리케이션은 보통 다음 세 가지 계층으로 구성됩니다.

프레젠테이션 계층 (Presentation Layer)
사용자가 직접 마주하는 부분입니다. 웹 브라우저에서 보낸 요청을 받고, 처리된 결과를 화면(웹 페이지)으로 보여주는 역할을 합니다. 컨트롤러와 뷰가 여기에 포함됩니다.

서비스 계층 (Service Layer)
애플리케이션의 핵심 로직이 담긴 곳입니다. 사용자의 요구사항을 실제로 어떻게 처리할지 결정하며, 프레젠테이션 계층과 데이터를 주고받는 다리 역할을 합니다.

퍼시스턴스 계층 (Persistence Layer)
데이터 액세스 계층이라고도 부릅니다. 데이터베이스에 직접 접근해서 데이터를 저장하거나 불러오는 일을 전담합니다.

3. 데이터 전달 모델 (Domain Object)
각 계층 사이에서 데이터를 실어 나르는 객체입니다. 예를 들어 도서 정보를 담는 Book 클래스 같은 것이 여기에 해당합니다.



4. 스프링 MVC의 구성 요소와 흐름
컨트롤러 (Controller)
사용자가 웹 주소를 입력하면 가장 먼저 반응하는 부분입니다. RequestMapping이라는 설정을 통해 특정 주소로 들어온 요청을 처리합니다.

모델 (Model)
컨트롤러가 작업한 결과물을 담아두는 바구니입니다. 이 바구니에 데이터를 담아서 뷰로 전달합니다.

뷰 (View)
사용자가 최종적으로 보게 되는 HTML 화면입니다. JSP 파일 등이 여기에 해당하며, 모델 바구니에 담긴 데이터를 꺼내서 화면에 뿌려줍니다.

5. 개발 순서 (권장 단계)
처음 개발을 시작할 때는 다음 순서를 참고하세요.

첫째, 데이터를 담을 도메인 객체(클래스)를 만듭니다.
둘째, 데이터베이스 처리를 담당할 퍼시스턴스 계층을 만듭니다.
셋째, 비즈니스 로직을 수행할 서비스 계층을 만듭니다.
넷째, 사용자의 요청을 받을 컨트롤러를 만듭니다.
다섯째, 마지막으로 화면을 보여줄 뷰(JSP 등)를 만듭니다.

6. 부가 기술: 부트스트랩 (Bootstrap)
강의 자료 후반부에 나오는 부트스트랩은 웹사이트 디자인을 쉽게 할 수 있게 도와주는 도구입니다. 복잡한 디자인 코드 없이도 버튼이나 메뉴 등을 예쁘게 만들 수 있고, 스마트폰이나 PC 등 기기 화면 크기에 맞춰 화면이 자동으로 조절되는 반응형 웹을 만드는 데 사용됩니다.
