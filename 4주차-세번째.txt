
Amazon EC2 강의교안 요약정리

1. Amazon EC2 개요
정의: AWS 클라우드에서 제공하는 가상 서버 서비스입니다.
특징:
- 원하는 만큼 가상 서버를 구축하고 보안 및 네트워킹을 구성할 수 있습니다.
- 수요에 따라 서버 용량을 조절하는 스케일링이 가능하여 효율적입니다.
- 인스턴스: 클라우드에서 실행되는 가상 컴퓨팅 환경을 부르는 단위입니다.

2. 주요 구성 요소
EC2를 실행하기 위해 필요한 4가지 핵심 요소입니다.

- AMI (Amazon Machine Image): 서버 실행에 필요한 운영체제(OS), 애플리케이션 등이 들어있는 템플릿입니다.
- 인스턴스 유형: 서버의 CPU, 메모리, 저장공간 등 성능을 결정하는 사양입니다.
- 키 페어 (Key Pair): 서버에 안전하게 접속하기 위한 암호화 키입니다. 공개 키는 AWS에, 개인 키는 사용자 PC에 저장됩니다.
- 보안 그룹 (Security Group): 인스턴스로 들어오고 나가는 트래픽을 제어하는 가상 방화벽입니다.

3. 인스턴스 유형 이름 읽는 법
인스턴스 이름(예: t3.micro)을 통해 성능을 알 수 있습니다.
- Family (t): 인스턴스의 목적 (범용, 컴퓨팅 최적화 등)
- Generation (3): 세대 번호 (숫자가 높을수록 최신 기술)
- Size (micro): 해당 유형 내에서의 상대적인 성능 규모

4. 스토리지 (EBS)
- Amazon EBS (Elastic Block Store): EC2 인스턴스에서 사용하는 가상 하드디스크입니다.
- 인스턴스가 중단되거나 종료되어도 데이터가 사라지지 않도록 영구적으로 보관할 수 있습니다.

5. 실습 단계 요약
1. AWS 콘솔 접속: 인스턴스 시작 버튼 클릭
2. 이름 설정: 서버의 이름 입력
3. AMI 선택: OS(예: Amazon Linux 2023) 선택
4. 인스턴스 유형 선택: 비용이 들지 않는 t3.micro(프리 티어) 권장
5. 키 페어 생성: .pem 형식의 키를 생성하여 내 컴퓨터에 안전하게 보관 (분실 시 접속 불가)
6. 네트워크 설정: 보안 그룹에서 SSH(22번 포트) 허용 확인
7. 인스턴스 실행: 생성 완료 후 실행 중 상태가 되면 접속 가능

주의사항
실습이 끝난 후에는 반드시 인스턴스를 종료(Terminate) 해야 추가 요금이 발생하지 않습니다. 중지(Stop) 상태에서도 스토리지 비용은 청구될 수 있으니 주의하세요.
