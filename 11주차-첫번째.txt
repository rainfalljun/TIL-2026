========================================
  Week 10 — 세션 2: 기획 & 일정 수립 요약
========================================

【오늘의 목표】
세션 종료 전까지 리포지토리에 아래 5개 파일 생성
  - .planning/00-vision.md       : 비전·목표
  - .planning/01-requirements.md : 요구사항
  - .planning/02-wbs.md          : Work Breakdown Structure
  - .planning/04-schedule.md     : 6주 일정 (10~14주차)
  - BONUS.md 초안                : 가산점 신청 트래킹

----------------------------------------
【기획 흐름 (AI Agent와 함께)】
  주제 한 줄
  → AI에게 "비전·문제정의" 요청
  → 사용자 시나리오 3개 도출
  → 요구사항 MoSCoW 분류
  → WBS 분해 (3단계)
  → 6주 일정으로 매핑
  → 위험 식별 + 대응

----------------------------------------
【WBS 작성 원칙】
  - 산출물을 명사형으로 표현 (예: "로그인 화면", "API 엔드포인트")
  - 동사형 활동 표현 금지 (예: "로그인을 만든다" ✗)
  - 최대 3단계 깊이까지만
  - 각 항목은 1~3일 단위로 추정 가능해야 함

----------------------------------------
【6주 일정 (권장 템플릿)】
  10주차 : 기획·일정         → .planning/* 작성 / 본인 리뷰
  11주차 : 설계·환경 구축    → architecture.md / Hello World 실행
  12주차 : 핵심 기능 1 + 중간발표 → 동작하는 프로토타입 데모
  13주차 : 핵심 기능 2 + 테스트  → 주요 기능 완성 / 테스트 통과
  14주차 : 마감·배포·문서    → 배포 산출물·문서 / 배포 링크 또는 빌드
  15주차 : 최종 발표         → 발표 자료 / 회고

----------------------------------------
【위험 식별 (최소 5개 필수)】
  카테고리     | 예시
  ------------|-------------------------------
  기술         | 처음 써보는 라이브러리 → 학습 시간 확보
  일정         | 시험·과제 폭주 주간
  협업 (2인)  | 의사결정 지연
  외부 의존    | API 키, 무료 티어 한도
  AI 의존      | AI가 만든 코드를 본인이 이해 못함

----------------------------------------
【AI Agent에게 시킬 5가지 메타 작업】
  1. 기획 문서 자동 생성
  2. WBS / 일정 자동 생성
  3. 회의록 / 결정 사항 자동 정리 (매 동기화 후)
  4. 일정 진행률 자동 보고서 (주 1회)
  5. 모르는 영역을 질문 목록으로 자동 추출

----------------------------------------
【의사결정 로그 (ADR)】
  - AI가 제시한 선택지 중 하나를 골랐다면 이유를 ADR로 기록
  - 파일 위치 예시:
      .planning/decisions/ADR-0001-mobile-framework.md
      .planning/decisions/ADR-0002-state-management.md
      .planning/decisions/ADR-0003-backend-choice.md
  - 각 ADR 구성: 배경 / 결정 / 대안 / 결과

----------------------------------------
【"이해했다"의 기준 체크리스트】
  [ ] 프로젝트의 한 줄 가치 제안을 말할 수 있다
  [ ] 첫 사용자와 사용 시나리오를 설명할 수 있다
  [ ] Must 기능 5개를 외워서 말할 수 있다
  [ ] 가장 큰 위험 1개와 대응 방법을 안다
  [ ] 이번 주 끝날 때까지 무엇이 완성되어야 하는지 안다
