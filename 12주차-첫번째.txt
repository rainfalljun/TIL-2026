================================================================================
01. 트리의 이해
================================================================================

■ 트리(Tree)란?
  - 원소들 간에 1:n 관계를 가지는 비선형 자료구조
  - 원소들 간에 계층관계를 가지는 계층형 자료구조(Hierarchical Data Structure)
  - 상위 원소에서 하위 원소로 내려가면서 확장되는 나무 모양의 구조

■ 트리의 주요 용어
  ┌──────────────┬──────────────────────────────────────────────────┐
  │ 용어          │ 설명                                             │
  ├──────────────┼──────────────────────────────────────────────────┤
  │ 노드(node)    │ 트리의 원소                                      │
  │ 루트 노드     │ 트리의 시작 노드 (레벨 0)                        │
  │ 간선(edge)    │ 노드를 연결하는 선 (부모-자식 관계 표현)         │
  │ 형제 노드     │ 같은 부모 노드의 자식 노드들                     │
  │ 조상 노드     │ 간선을 따라 루트 노드까지 경로에 있는 모든 노드  │
  │ 자손 노드     │ 서브 트리에 있는 하위 레벨의 노드들              │
  │ 서브 트리     │ 부모 노드와 연결된 간선을 끊었을 때 생성되는 트리│
  │ 차수(degree)  │ 노드에 연결된 자식 노드의 수                     │
  │ 단말 노드     │ 차수가 0인 노드 (리프 노드)                      │
  │ 높이(height)  │ 루트에서 노드에 이르는 간선의 수 (레벨)          │
  │ 포리스트      │ 서브 트리의 집합                                 │
  └──────────────┴──────────────────────────────────────────────────┘

02. 이진 트리 (Binary Tree)
================================================================================

■ 개념
  - 트리의 모든 노드의 차수를 2 이하로 제한한 트리
  - 모든 노드는 왼쪽 자식 노드와 오른쪽 자식 노드만 가짐
  - 공백 노드도 자식 노드로 취급 (0 ≤ 노드의 차수 ≤ 2)
  - 순환적으로 구성됨 (왼쪽/오른쪽 서브 트리도 이진 트리)

■ 이진 트리의 특성
  - 노드가 n개인 이진 트리의 간선 수 = (n-1)개
  - 높이가 h인 이진 트리의 노드 개수: 최소 (h+1)개, 최대 (2^(h+1) - 1)개

■ 이진 트리의 종류
  ① 포화 이진 트리 (Full Binary Tree)
     - 모든 레벨에 노드가 포화 상태로 차 있는 이진 트리
     - 높이 h일 때, 노드 수 = 2^(h+1) - 1

  ② 완전 이진 트리 (Complete Binary Tree)
     - 마지막 레벨을 제외한 모든 레벨이 완전히 채워진 이진 트리
     - 마지막 레벨의 노드들은 왼쪽부터 차례대로 채워짐

  ③ 편향 이진 트리 (Skewed Binary Tree)
     - 높이 h일 때 h+1개의 노드를 가지며
     - 모든 노드가 왼쪽 또는 오른쪽 한 방향으로만 서브 트리를 가짐

================================================================================
03. 이진 트리의 구현
================================================================================

■ 순차 자료구조를 이용한 구현
  - 1차원 배열 사용 (인덱스 0번은 비워 둠, 1번에 루트 저장)
  - 노드 i의 부모 노드 인덱스 = └i/2┘
  - 노드 i의 왼쪽 자식 인덱스 = 2i
  - 노드 i의 오른쪽 자식 인덱스 = 2i + 1
  ※ 편향 이진 트리의 경우 배열 낭비가 심함

■ 연결 자료구조를 이용한 구현
  - 각 노드: 데이터 필드 + 왼쪽 링크 필드 + 오른쪽 링크 필드
  - 자식 노드가 없으면 링크 필드에 NULL 저장

================================================================================
04. 이진 트리의 순회 (Traversal)
================================================================================

■ 순회의 개념
  - 모든 원소를 빠트리거나 중복하지 않고 처리하는 연산
  - 세 가지 기본 작업: D(현재 노드 방문), L(왼쪽 서브 트리 이동), R(오른쪽 서브 트리 이동)
  - 왼쪽 서브 트리를 오른쪽 서브 트리보다 먼저 수행

■ 순회의 종류
  ① 전위 순회 (Preorder Traversal) : D → L → R
     - 현재 노드를 가장 먼저 방문
     - 수식 트리를 전위 순회하면 전위 표기식(prefix) 생성
     - 예) A-B-D-H-E-I-J-C-F-G-K

  ② 중위 순회 (Inorder Traversal) : L → D → R
     - 현재 노드를 중간에 방문
     - 수식 트리를 중위 순회하면 중위 표기식(infix) 생성
     - 예) H-D-B-I-E-J-A-F-C-G-K

  ③ 후위 순회 (Postorder Traversal) : L → R → D
     - 현재 노드를 가장 마지막에 방문
     - 수식 트리를 후위 순회하면 후위 표기식(postfix) 생성
     - 예) H-D-I-J-E-B-F-K-G-C-A
     - 응용: 폴더 전체 용량 계산 (하위 폴더 먼저 계산)

■ 스레드 이진 트리 (Thread Binary Tree)
  - 재귀호출 없이 순회할 수 있도록 수정한 이진 트리
  - 자식 노드가 없는 링크 필드를 순회 순서상의 다른 노드를 가리키도록 설정
  - 왼쪽 스레드: 선행자(predecessor), 오른쪽 스레드: 후속자(successor) 저장
  - isThread 필드를 추가하여 링크 필드와 스레드 구분

05. 이진 탐색 트리 (Binary Search Tree)
================================================================================

■ 개념
  - 이진 트리를 탐색용 자료구조로 사용하기 위해 노드 위치를 정의한 것
  - 규칙: 왼쪽 서브 트리 < 부모 노드 < 오른쪽 서브 트리

■ 탐색 연산
  ① 루트에서 탐색 시작
  ② 찾을 키값 x와 현재 노드의 키값을 비교
     - x = 현재 키값 → 탐색 성공
     - x < 현재 키값 → 왼쪽 서브 트리 탐색
     - x > 현재 키값 → 오른쪽 서브 트리 탐색
  ③ 순환적으로 반복

■ 삽입 연산
  ① 탐색 연산 수행 (같은 원소가 있으면 삽입 불가)
  ② 탐색 실패한 위치에 원소 삽입

■ 삭제 연산
  ① 탐색 연산으로 삭제할 노드 위치 파악
  ② 삭제 후 이진 탐색 트리 재구성
     - 단말 노드(차수=0): 그냥 삭제
     - 자식 1개(차수=1): 삭제 후 자식 노드가 부모 자리를 물려받음
     - 자식 2개(차수=2): 후계자 노드 선택 후 자리 물려줌
       · 후계자 선택 방법 1: 왼쪽 서브 트리에서 가장 큰 노드
       · 후계자 선택 방법 2: 오른쪽 서브 트리에서 가장 작은 노드

06. 균형 이진 탐색 트리 - AVL 트리
================================================================================

■ 개념
  - Adelson-Velskii, Landis Tree (대표적인 균형 이진 탐색 트리)
  - 각 노드에서 왼쪽/오른쪽 서브 트리 높이 차이가 1 이하인 트리
  - 균형 인수(BF, Balance Factor) = hL - hR (각 노드에서 {-1, 0, +1} 값만 허용)

■ 불균형 유형 및 회전 연산
  ┌──────┬──────────────────────────────────────────────────────────┐
  │ 유형 │ 설명 및 회전 방법                                         │
  ├──────┼──────────────────────────────────────────────────────────┤
  │ LL   │ 왼쪽-왼쪽 유형. BF=+2, 왼쪽 자식 BF=+ → LL 회전(단순)   │
  │ RR   │ 오른쪽-오른쪽 유형. BF=-2, 오른쪽 자식 BF=- → RR 회전   │
  │ LR   │ 왼쪽-오른쪽 유형. BF=+2, 왼쪽 자식 BF=- → RR 후 LL 회전│
  │ RL   │ 오른쪽-왼쪽 유형. BF=-2, 오른쪽 자식 BF=+ → LL 후 RR 회전│
  └──────┴──────────────────────────────────────────────────────────┘
  - 단순 회전(Single Rotation): LL, RR (한 번 회전)
  - 이중 회전(Double Rotation): LR, RL (두 번 회전)

■ AVL 트리의 특징
  - 삽입·삭제 후 불균형 발생 시 회전 연산으로 균형 복구
  - 일반 이진 탐색 트리보다 탐색 효율이 높음 (항상 균형 유지)

07. 히프 (Heap)
================================================================================

■ 개념
  - 완전 이진 트리에서 키값이 가장 크거나 작은 노드를 빠르게 찾기 위한 자료구조
  - 최대 히프(Max Heap): 부모 노드의 키값 ≥ 자식 노드의 키값 (루트 = 최댓값)
  - 최소 히프(Min Heap): 부모 노드의 키값 ≤ 자식 노드의 키값 (루트 = 최솟값)

■ 삽입 연산
  ① 완전 이진 트리 조건을 만족하도록 다음 자리(n+1번)에 노드 확장 및 임시 저장
  ② 부모 노드와 크기 비교
     - 삽입 원소 > 부모 노드 → 부모와 자리 교환 (위로 이동)
     - 삽입 원소 ≤ 부모 노드 → 현재 위치 확정
  ③ 루트까지 반복

■ 삭제 연산
  - 루트 노드만 삭제 가능
  ① 루트 노드의 원소 삭제 및 반환
  ② 마지막 노드(n번)를 삭제하고 해당 원소를 루트 자리에 임시 저장
  ③ 자식 노드와 크기 비교하면서 제자리 찾기
     - 임시 원소 < 자식 노드 중 큰 값 → 자식과 자리 교환 (아래로 이동)
     - 임시 원소 ≥ 모든 자식 노드 → 현재 위치 확정

■ 히프의 구현
  - 1차원 배열(순차 자료구조)로 구현
  - 인덱스 관계를 이용해 부모/자식 노드 접근 용이
    · 부모 노드: └i/2┘, 왼쪽 자식: 2i, 오른쪽 자식: 2i+1
