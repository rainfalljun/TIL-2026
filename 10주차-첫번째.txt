========================================
       AWS ELB (Elastic Load Balancer) 요약정리
========================================

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. ELB란?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- AWS에서 제공하는 트래픽 분산 서비스
- 들어오는 애플리케이션 트래픽을 EC2, 컨테이너, IP 주소, Lambda 등
  여러 대상에 자동으로 분산시킴
- 단일 가용영역 및 여러 가용영역에서 다양한 부하 처리 가능

[ELB가 필요한 이유]
- EC2가 많을 경우 사용자가 모든 IP를 알고 관리하는 것이 불가능
- ELB 도메인 하나로 여러 EC2에 트래픽을 자동 분산

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
2. ELB 주요 특징
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 다수의 서비스에 트래픽 분산
- Health Check: 트래픽을 발생시켜 인스턴스 생존 여부 확인
- Auto Scaling과 연동 가능
- 여러 가용영역에 분산 가능
- IP 주소가 지속적으로 변경되어 고정 불가 → 항상 도메인 기반으로 사용
- 고가용성, 자동 확장/축소, 강력한 보안 제공

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3. ELB 종류 (3가지)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
① Application Load Balancer (ALB) - "똑똑함"
  - HTTP/HTTPS 트래픽 처리
  - 트래픽을 모니터링하여 라우팅 가능
  - 경로/도메인 기반 라우팅 (예: image.test.com → 이미지 서버)
  - 마이크로서비스 및 컨테이너 기반 아키텍처에 적합

② Network Load Balancer (NLB) - "빠름"
  - TCP 기반 빠른 트래픽 분산
  - Elastic IP 할당 가능

③ Classic Load Balancer (CLB) - "구버전"
  - 현재는 거의 사용하지 않음

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
4. ALB Target Group (대상 그룹)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- ALB가 트래픽을 분산시킬 대상을 논리적으로 묶어놓은 그룹
- 대상 유형: EC2 인스턴스, IP 주소, Lambda 함수, ELB
- 동작 방식:
  ALB → 라우팅 규칙에 따라 타겟 그룹으로 요청 전달
       → 헬스체크 통과한 대상에게 트래픽 전달
- 타겟 그룹별로 포트, 프로토콜, 헬스체크 설정 가능

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
5. AWS Lambda란?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 서버를 직접 관리하지 않고 코드를 실행할 수 있는 서비스 (서버리스)
- 서버 불필요: EC2처럼 서버를 띄우고 관리할 필요 없음
- 초 단위 과금: 실제 코드 실행 시간만 비용 지불
- 자동 확장: 호출이 많아지면 자동으로 인스턴스 확장
- 이벤트 기반 실행: S3, DynamoDB, API Gateway 등 다양한 이벤트로 동작
- 지원 언어: Python, Node.js, Java, Go 등

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
6. 실습 요약
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[실습 1] ALB 생성
  1) 시작 템플릿 수정 (User Data에 인스턴스 ID 출력 스크립트 추가)
  2) 시작 템플릿으로 EC2 2대 생성 (ALB1, ALB2)
  3) 대상 그룹(Target Group) 생성 → EC2 2대 등록
  4) ALB 생성 → 리스너 포트 80 → 대상 그룹 연결
  5) DNS 주소로 접속 확인 (새로고침 시 다른 인스턴스 ID 출력)

[실습 2] 경로기반 리디렉션 (ALB → S3)
  1) S3 버킷 정적 웹사이트 호스팅 활성화 (인덱스 문서: index.html)
  2) ALB 리스너 규칙 추가
     - 조건: 경로 = /s3
     - 작업: URL 리디렉션 → S3 엔드포인트
     - 상태코드: 302
  3) [ALB 도메인]/s3 접속 시 S3 index.html로 이동 확인

[실습 3] Lambda 연동
  1) Lambda 함수 생성 (Python 3.14, "Hello Lambda" JSON 반환)
  2) Lambda 대상 그룹 생성
  3) ALB 리스너 규칙 추가
     - 조건: 경로 = /lambda
     - 작업: Lambda 대상 그룹으로 전달
  4) [ALB 도메인]/lambda 접속 시 {"message": "Hello Lambda"} 확인

========================================
             핵심 정리
========================================
ELB = 트래픽 분산 + Health Check + Auto Scaling 연동
ALB = 경로/도메인 기반 스마트 라우팅
NLB = 고성능 TCP 트래픽 처리
Lambda = 서버리스 코드 실행 (이벤트 기반, 초 단위 과금)
========================================
