# Chapter 05 스택 강의 요약

## 1. 스택의 이해

**스택(Stack)이란?**
접시를 쌓듯이 자료를 차곡차곡 쌓아 올린 형태의 자료구조로, **후입선출(LIFO, Last-In-First-Out)** 구조입니다. 저장된 원소는 top으로 정한 곳에서만 삽입·삭제가 가능합니다.

**핵심 연산**
- **push**: top에 원소를 삽입. top이 stack_SIZE 초과 시 → overflow 오류
- **pop**: top에서 원소를 삭제 후 반환. 공백 스택에서 pop 시 → underflow 오류
- **peek**: top 원소를 삭제 없이 반환

---

## 2. 스택의 구현

| 구분 | 순차 자료구조 (배열) | 연결 자료구조 (연결 리스트) |
|------|------|------|
| 구조 | 1차원 배열 | 단순 연결 리스트 |
| top 초기값 | -1 | null |
| 포화 상태 | top = n-1 | 없음 |
| 장점 | 구현이 단순함 | 크기가 유동적 |
| 단점 | 크기 변경 어려움 | 포인터 관리 필요 |

---

## 3. 스택의 응용

**① 역순 문자열 만들기**
문자열을 순서대로 push → 순서대로 pop하면 역순 문자열 완성

**② 시스템 스택**
함수 호출 시 스택 프레임에 지역변수·매개변수·복귀주소를 저장하여 함수 실행 순서 관리. 함수가 종료되면 스택을 pop하여 이전 함수로 복귀

**③ 수식의 괄호 검사**
- 왼쪽 괄호 → push
- 오른쪽 괄호 → pop 후 같은 종류인지 확인
- 수식 종료 시 스택이 공백이면 올바른 수식

**④ 수식 표기법 변환 (중위 → 후위)**

| 표기법 | 설명 | 예시 |
|------|------|------|
| 중위(infix) | 연산자가 피연산자 사이 | A+B |
| 전위(prefix) | 연산자가 피연산자 앞 | +AB |
| 후위(postfix) | 연산자가 피연산자 뒤 | AB+ |

중위 → 후위 변환 규칙: 피연산자는 바로 출력, 연산자는 스택에 push, 오른쪽 괄호를 만나면 pop하여 출력, 수식 끝나면 스택을 모두 pop

**⑤ 후위 표기법 수식 연산**
- 피연산자 → push
- 연산자 → 피연산자 2개를 pop하여 연산 후 결과를 다시 push
- 수식 종료 → 스택에서 pop하여 최종 결과 출력
