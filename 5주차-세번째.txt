스프링 웹 MVC 컨트롤러 핵심 요약

1. 컨트롤러(Controller) 개요
- 정의: 웹 브라우저의 요청을 처리하고 결과를 뷰에 전달하는 스프링 빈 클래스입니다.
- 역할: MVC 패턴에서 사용자 요청에 대응하는 비즈니스 로직을 수행합니다.
- 흐름: 클라이언트 요청이 들어오면 디스패처서블릿이 이를 받아 적절한 컨트롤러로 연결합니다.

2. 컨트롤러 정의 및 등록
- 정의 방법: 클래스 상단에 @Controller 어노테이션을 선언하여 만듭니다.
- 자동 등록: 스프링 설정 파일에 context:component-scan을 설정하면 컨트롤러를 자동으로 찾아 빈으로 등록합니다.
- 스캔 대상: @Component, @Repository, @Service, @Controller, @RestController 등이 포함됩니다.

3. @RequestMapping (요청 경로 매핑)
- 기능: 클라이언트의 URL 요청과 이를 처리할 메소드를 연결합니다.
- 적용 대상: 클래스 수준(공통 경로)과 메소드 수준(세부 경로)에 모두 사용할 수 있습니다.
- 주요 속성: value는 매핑할 경로를 의미하며, method는 GET이나 POST 같은 요청 방식을 지정합니다.
- 경로 결합: 클래스에 지정한 경로와 메소드에 지정한 경로가 합쳐져 최종 접속 URL이 됩니다.

4. 요청 방식에 따른 단순화 어노테이션
- @GetMapping: GET 방식 매핑 (데이터 조회 시 사용)
- @PostMapping: POST 방식 매핑 (데이터 생성 시 사용)
- @PutMapping: PUT 방식 매핑 (데이터 수정 시 사용)
- @DeleteMapping: DELETE 방식 매핑 (데이터 삭제 시 사용)

5. 모델(Model) 인터페이스
- 역할: 처리 결과를 뷰에 전달하기 위해 데이터를 담아두는 객체입니다.
- Model: addAttribute() 메소드를 사용하여 데이터를 저장하고 전달합니다.
- ModelMap: Map 인터페이스 기능을 활용해 데이터를 관리합니다.
- ModelAndView: 데이터(Model)와 이동할 페이지 정보(View)를 하나의 객체로 묶어서 관리합니다.
